01 / What it does
What the skill actually ships
The README positions the skill as a zero-dependency presentation generator with a persistent browser editor. The deck it creates is meant to stay useful after generation, not freeze into a one-off artifact.
GenerateSingle fileInline CSS and JS, style previews, and no build step.
EditLayout runtimeDrag objects, resize text blocks, multi-select, and keep undo/redo inside the file.
DeliverPersist or exportSave the full deck structure to localStorage during iteration, then export a sanitized standalone HTML when the layout is ready to ship.
02 / Why the fork exists
The trade-off is deliberate
The editable fork keeps the original deck-generation path, but adds weight on purpose: a Pages sidebar, object editing, history, persistence, and export. The goal is not the smallest file. The goal is iterative control.
Parent skillRead-only firstUse it when the deck is close to final and file weight matters more than in-browser iteration.
Editable forkDraft to reviewUse it when layouts keep moving and the browser file itself needs to stay editable after generation.
Better for review cyclesHeavier, but more forgiving

04 / Editing runtime
The browser file keeps doing work
Editable mode is more than text editing. The runtime is responsible for object manipulation, focus routing, history, Pages filmstrip behavior, and storage of the whole deck structure.

06 / Contracts worth protecting
The runtime only feels stable when the boring rules hold
The editable skill is sensitive to a few structural guarantees. Miss any one of them and navigation, filmstrip behavior, persistence, or editing affordances start drifting out of sync.
SlidesStable ids`section.slide#slide-N` is the persistence anchor. Reordering depends on it.
ObjectsUnique `data-oid`Selections, patches, deletes, and saves all assume object ids are unique across the whole deck.
QueriesRoot-scoped listsFilmstrip clones look like slides, so slide enumeration must stay scoped to `.slides-offset`.
StylingViewport + chrome tokensThe slide theme and edit chrome must both stay readable while every slide still fits the viewport.
